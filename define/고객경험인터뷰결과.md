# 고객경험 인터뷰 결과

## 인터뷰 개요
- **목적**: 업무 목적에 적합한 생성형 AI를 빠르게 찾고 싶은 직장인의 현재 경험 분석
- **대상**: 직장인 5명
- **기간**: 2024년 9월
- **방법**: 1:1 심층 인터뷰 (각 60분)

---

## 인터뷰 대상자 프로필

### 1. 김민수 (32세, 마케팅 담당자)
- **소속**: 중견 IT기업 마케팅팀
- **경력**: 7년차
- **AI 활용 목적**: 콘텐츠 제작, 캠페인 기획

### 2. 이정현 (38세, 개발팀 리더)
- **소속**: 스타트업 CTO
- **경력**: 12년차
- **AI 활용 목적**: 코드 리뷰, 문서화, 아키텍처 설계

### 3. 박소영 (28세, HR 담당자)
- **소속**: 대기업 인사팀
- **경력**: 4년차
- **AI 활용 목적**: 채용 공고 작성, 교육 자료 제작

### 4. 최대한 (41세, 영업 매니저)
- **소속**: 제조업체 영업부
- **경력**: 15년차
- **AI 활용 목적**: 제안서 작성, 고객 대응

### 5. 송혜진 (34세, 기획자)
- **소속**: 금융권 전략기획팀
- **경력**: 9년차
- **AI 활용 목적**: 시장 분석, 보고서 작성

**부정적 느낌**
- 어떤 것을 선택해야 할지 막막함

**생각**
- "개발 언어별로 성능 차이가 있을까?"
- "코드 품질과 보안성을 어떻게 검증하지?"
- "팀 전체 도입 시 비용 대비 효과는?"

**긍정적 느낌**
- 실제 코딩에서 확실한 도움이 됨을 확인
- 다양한 프로그래밍 언어 지원

**부정적 느낌**
- 도구별 정확한 차이점 파악의 어려움
- 보안 및 라이선스 정책의 복잡성
- 성능 평가 기준의 모호함

## 종합 분석

### 주요 페인 포인트
1. **도구 선택의 어려움**: 너무 많은 옵션으로 인한 혼란
2. **학습 비용**: 새로운 도구 익히는 데 소요되는 시간과 노력
3. **비용 부담**: 개인 또는 팀 단위의 구독 비용 부담
4. **보안 및 정책**: 회사 정책과 개인정보 보호에 대한 우려
5. **성능 검증**: AI 생성 결과의 정확성과 신뢰성 검증 필요
6. **전문성 한계**: 업무 도메인별 특화 기능 부족

### 핵심 니즈
1. **빠른 비교 선택**: 업무 목적에 맞는 도구를 빠르게 찾고 비교할 수 있는 서비스
2. **검증된 정보**: 실제 사용자 후기와 객관적인 성능 평가 정보
3. **학습 지원**: 효과적인 활용법과 베스트 프랙티스 가이드
4. **비용 효율성**: 투자 대비 효과를 명확히 알 수 있는 정보
5. **보안 가이드**: 안전한 사용을 위한 가이드라인과 정책 정보
6. **지속적 업데이트**: 빠르게 변화하는 AI 도구 트렌드에 대한 지속적인 정보 제공

### 서비스 개발 시사점
1. **개인화된 추천**: 직무, 업무 특성, 예산에 맞는 맞춤형 도구 추천
2. **실사용자 리뷰**: 실제 업무 환경에서의 활용 사례와 효과성 정보
3. **학습 리소스**: 단계별 학습 가이드와 프롬프트 템플릿 제공
4. **ROI 계산기**: 투자 대비 효과를 예측할 수 있는 도구
5. **정책 가이드**: 기업별 도입 가이드라인과 보안 체크리스트
6. **트렌드 모니터링**: 새로운 도구 출시와 업데이트 정보의 실시간 제공
