# 생성형 AI 정보제공 서비스 - UI/UX 설계서

## 개요

### 목적
'업무 목적에 적합한 생성형 AI를 빠르게 찾고 싶은 직장인'을 위한 생성형AI 정보제공 서비스의 사용자 인터페이스 및 사용자 경험 설계

### 설계 원칙
- **빠른 발견**: 최소 클릭으로 원하는 AI 도구 발견
- **직관적 사용**: 별도 학습 없이 바로 사용 가능한 인터페이스
- **신뢰할 수 있는 추천**: 명확한 추천 근거와 투명한 정보 제공
- **개인화 경험**: 사용자의 업무 상황에 맞춤화된 경험

## 화면 간 사용자 플로우 정의

### 주요 플로우 (Happy Path)
```
시작 → 로그인 → 메인대시보드 → 업무상황입력 → 추천결과 → 도구상세보기 → 만족도평가 → 완료
```

### 보조 플로우
1. **신규 가입**: 시작 → 회원가입 → 로그인 → 메인대시보드
2. **재방문**: 로그인 → 메인대시보드 → 이전 추천 이력 확인
3. **프로필 관리**: 메인대시보드 → 프로필관리 → 메인대시보드
4. **추천 재시작**: 추천결과 → 업무상황입력 (수정) → 추천결과

### 플로우 분기점
- **로그인 분기**: 기존 사용자 vs 신규 사용자
- **추천 후 분기**: 만족 vs 불만족 vs 추가 탐색
- **재방문 분기**: 새로운 추천 vs 이전 추천 확인

## 화면별 상세 설계

### 01-로그인

#### 개요
- **목적**: 사용자 인증을 통한 서비스 접근
- **관련 유저스토리**: UFR-USER-010
- **비즈니스 중요도**: 필수 (게이트키퍼 역할)

#### 주요 기능
- 사용자 ID/비밀번호 입력
- 로그인 상태 유지 옵션
- 회원가입 링크 제공
- 소셜 로그인 지원 (선택적)

#### UI 구성요소
- **헤더**: 서비스 로고, 브랜드명
- **로그인 폼**:
  - 이메일/ID 입력 필드
  - 비밀번호 입력 필드 (보안 마스킹)
  - "로그인 유지" 체크박스
  - "로그인" 버튼 (Primary CTA)
- **보조 액션**: 
  - "회원가입" 링크 (Secondary CTA)
  - "비밀번호 찾기" 링크
- **푸터**: 서비스 소개 간단 설명

#### 인터랙션
- 입력 필드 포커스 시 테두리 강조
- 필수 입력 누락 시 실시간 검증 메시지
- 로그인 실패 시 명확한 오류 메시지
- 성공 시 메인대시보드로 자동 이동

### 02-회원가입

#### 개요
- **목적**: 신규 사용자 계정 생성 및 기본 정보 수집
- **관련 유저스토리**: UFR-USER-020
- **비즈니스 중요도**: 필수 (사용자 확보)

#### 주요 기능
- 단계별 정보 입력 (2단계 구성)
- 실시간 입력 검증
- 중복 계정 확인
- 직무 정보 수집

#### UI 구성요소
- **진행 표시**: 단계 인디케이터 (1/2, 2/2)
- **1단계 - 기본 정보**:
  - 이름 입력 필드
  - 이메일 입력 필드 (중복 확인 버튼)
  - 비밀번호 입력 필드 (강도 표시)
  - 비밀번호 확인 필드
- **2단계 - 직무 정보**:
  - 소속 입력 필드 (자동완성)
  - 직무 선택 드롭다운
  - 경력 선택 라디오 버튼 (신입/1-3년/3-7년/7년+)
- **액션 버튼**:
  - "다음" / "이전" 버튼
  - "가입 완료" 버튼 (Primary CTA)

#### 인터랙션
- 단계별 유효성 검증 후 진행
- 이메일 중복 확인 비동기 처리
- 비밀번호 강도 실시간 표시
- 가입 완료 후 로그인 화면으로 이동

### 03-메인대시보드

#### 개요
- **목적**: 서비스 홈페이지, 주요 액션 시작점
- **관련 유저스토리**: UFR-WEB-010
- **비즈니스 중요도**: 필수 (서비스 중심)

#### 주요 기능
- 서비스 소개 및 시작 가이드
- 최근 추천 이력 표시
- 인기 AI 도구 미리보기
- 빠른 액션 버튼들

#### UI 구성요소
- **헤더**: 
  - 서비스 로고 및 네비게이션
  - 사용자 프로필 드롭다운
- **메인 섹션**:
  - Hero 영역: 서비스 소개 및 "AI 도구 찾기 시작" CTA
  - 최근 추천 이력 카드 (최대 3개)
  - 인기 AI 도구 섹션 (최대 6개)
- **사이드바**:
  - 빠른 액션 메뉴
  - 사용 통계 (추천 받은 횟수, 평가한 도구 수)
- **푸터**: 추가 링크 및 고객지원

#### 인터랙션
- Hero CTA 클릭 시 업무상황입력으로 이동
- 최근 이력 클릭 시 해당 추천결과로 이동
- 인기 도구 클릭 시 도구상세보기로 이동
- 부드러운 스크롤 애니메이션

### 04-업무상황입력

#### 개요
- **목적**: AI 도구 추천을 위한 사용자 업무 상황 수집
- **관련 유저스토리**: UFR-ANAL-010, UFR-ANAL-020
- **비즈니스 중요도**: 핵심 (추천 품질의 기초)

#### 주요 기능
- 6가지 필수 정보 수집 (업무유형, 산업분야, 팀규모, 예산범위, 긴급도, 기술수준)
- 실시간 입력 완성도 표시
- 스마트 추천 (입력 기반 자동완성)
- 최소 3개 이상 입력 강제

#### UI 구성요소
- **헤더**: 진행률 표시 (완성도 %)
- **입력 섹션들**:
  - **업무유형**: 태그 선택 방식 (다중 선택)
  - **산업분야**: 드롭다운 + 검색
  - **팀규모**: 슬라이더 + 직접 입력
  - **예산범위**: 범위 선택 슬라이더
  - **긴급도**: 5단계 라디오 버튼 (시급함-여유있음)
  - **기술수준**: 3단계 선택 (초급-중급-고급)
- **사이드 정보**: 
  - 완성도 진행바
  - 입력 팁 및 예시
- **액션 버튼**: "추천받기" (완성도 60% 이상 시 활성화)

#### 인터랙션
- 각 섹션 입력 시 완성도 실시간 업데이트
- 미입력 필수 항목 강조 표시
- 입력 완료 시 자동 포커스 이동
- 추천받기 클릭 시 로딩 애니메이션과 함께 추천결과로 이동

### 05-추천결과

#### 개요
- **목적**: 분석된 업무 상황에 맞는 AI 도구 추천 목록 제시
- **관련 유저스토리**: UFR-RECO-010, UFR-WEB-020
- **비즈니스 중요도**: 핵심 (서비스 핵심 가치)

#### 주요 기능
- 최대 5개 AI 도구 추천 표시
- 무료/유료 구분 표시
- 사용 난이도 레벨 표시
- 정렬 및 필터 기능
- 추천 이유 설명

#### UI 구성요소
- **헤더**: 
  - 추천 요약 (입력한 조건 요약)
  - 결과 개수 및 정렬 옵션
- **필터 바**: 
  - 가격 필터 (전체/무료/유료)
  - 난이도 필터 (초급/중급/고급)
  - 카테고리 필터
- **추천 카드 리스트**:
  - 도구 아이콘/로고
  - 도구명 및 개발사
  - 가격 정보 (무료/유료 라벨)
  - 난이도 레벨 (★☆☆ 형태)
  - 추천 이유 요약 (2-3줄)
  - 신뢰도 점수 (%로 표시)
  - "상세보기" 버튼
- **사이드 정보**: 
  - 추천 기준 설명
  - "다시 찾기" 버튼

#### 인터랙션
- 카드 hover 시 추가 정보 툴팁 표시
- 정렬/필터 변경 시 부드러운 재배치 애니메이션
- 무한 스크롤 (추가 추천 있는 경우)
- 상세보기 클릭 시 도구상세보기로 이동

### 06-도구상세보기

#### 개요
- **목적**: 선택된 AI 도구의 상세 정보 제공 및 의사결정 지원
- **관련 유저스토리**: UFR-TOOL-030, UFR-WEB-030
- **비즈니스 중요도**: 필수 (구매 결정 지원)

#### 주요 기능
- 도구 상세 정보 표시
- 가격 플랜 비교
- 사용자 리뷰 및 평점
- 유사 도구 비교
- 외부 링크 연결

#### UI 구성요소
- **헤더**: 
  - 도구명 및 개발사
  - 전체 평점 (별점 + 수치)
  - 가격 라벨 및 "사용해보기" CTA
- **탭 구성**:
  - **개요 탭**: 주요 기능, 특징, 스크린샷
  - **가격 탭**: 플랜별 비교 테이블
  - **리뷰 탭**: 사용자 리뷰 목록 및 통계
  - **비교 탭**: 유사 도구와의 비교표
- **사이드바**:
  - 기본 정보 (카테고리, 난이도, 지원 언어)
  - 빠른 액세스 링크
  - 추천 이유 상세 설명
- **하단**: 
  - 관련 도구 추천 (3-4개)
  - "만족도 평가하기" 버튼

#### 인터랙션
- 탭 전환 시 부드러운 페이드 애니메이션
- 스크린샷 클릭 시 확대 모달
- 외부 링크는 새 창에서 열기
- 스크롤 진행에 따른 사이드바 고정

### 07-만족도평가

#### 개요
- **목적**: 추천 품질 개선을 위한 사용자 피드백 수집
- **관련 유저스토리**: UFR-EVAL-010
- **비즈니스 중요도**: 필수 (서비스 개선)

#### 주요 기능
- 5점 척도 만족도 평가
- 선택적 상세 피드백
- 개선 제안 수집
- 3점 이하 시 추가 질문

#### UI 구성요소
- **평가 섹션**:
  - 추천받은 도구 요약 표시
  - 별점 선택 UI (1-5점)
  - 만족도 설명 텍스트 (동적 변경)
- **피드백 섹션**:
  - "어떤 점이 좋았나요?" (체크박스 다중 선택)
  - "개선이 필요한 부분" (3점 이하 시 표시)
  - 자유 의견 텍스트 영역
- **완료 섹션**:
  - "평가 완료" 버튼
  - "메인으로 돌아가기" 링크

#### 인터랙션
- 별점 선택 시 실시간 피드백 메시지 변경
- 3점 이하 선택 시 추가 필드 슬라이드 인
- 평가 완료 시 감사 메시지 표시
- 자동으로 메인대시보드 이동

### 08-프로필관리

#### 개요
- **목적**: 사용자 정보 관리 및 개인화 설정
- **관련 유저스토리**: UFR-USER-030
- **비즈니스 중요도**: 보완 (개인화 향상)

#### 주요 기능
- 기본 정보 수정
- 직무 정보 업데이트
- 과거 이용 이력 조회
- 개인화 설정

#### UI 구성요소
- **탭 네비게이션**:
  - **기본정보**: 이름, 이메일, 비밀번호 변경
  - **직무정보**: 소속, 직무, 경력 업데이트
  - **이용이력**: 추천 이력, 평가 이력 조회
  - **설정**: 알림 설정, 개인화 옵션
- **편집 폼**: 각 탭별 해당 정보 편집 양식
- **저장 버튼**: 변경사항 저장 CTA

#### 인터랙션
- 실시간 입력 검증
- 변경사항 있을 시 저장 버튼 활성화
- 저장 완료 시 토스트 메시지
- 민감 정보 수정 시 재인증 요구

## 화면간 전환 및 네비게이션

### 글로벌 네비게이션
- **헤더 네비게이션**: 로고(홈으로), 메인메뉴, 사용자 프로필
- **브레드크럼**: 현재 위치 표시 (3depth 이상 시)
- **사이드 네비게이션**: 모바일에서 햄버거 메뉴로 접근

### 상태별 네비게이션 구조
```
├── 미인증 상태
│   ├── 로그인
│   └── 회원가입
└── 인증 상태  
    ├── 메인대시보드 (홈)
    ├── 업무상황입력
    ├── 추천결과
    ├── 도구상세보기
    ├── 만족도평가
    └── 프로필관리
```

### 전환 규칙
1. **순차 전환**: 업무상황입력 → 추천결과 → 도구상세보기 → 만족도평가
2. **자유 전환**: 메인대시보드에서 모든 화면 접근 가능
3. **제한 전환**: 미완료 프로세스는 중간 진입 제한

### 애니메이션 가이드
- **페이지 전환**: 0.3s ease-in-out 페이드
- **모달/팝업**: 0.2s 스케일 + 페이드
- **로딩**: 스켈레톤 UI 또는 스피너

## 반응형 설계 전략

### 브레이크포인트
- **Desktop**: 1200px 이상
- **Tablet**: 768px - 1199px  
- **Mobile**: 767px 이하

### 적응형 레이아웃
- **Desktop**: 사이드바 + 메인 컨텐츠 (2-3컬럼)

- **Mobile**: 단일 컬럼, 하단 네비게이션

1. **업무상황입력**: 한 번에 하나씩 입력하는 스텝 형태
2. **추천결과**: 카드를 세로 스택으로 배치
3. **도구상세보기**: 탭을 아코디언으로 변경

## 성능 최적화 방안

### 로딩 성능
- **Critical CSS**: 첫 화면 렌더링 최적화
- **Code Splitting**: 라우트별 번들 분할

- **Progressive Loading**: 단계적 컨텐츠 로드
- **Error Boundary**: 에러 상황 우아한 처리
- **Offline Support**: 네트워크 단절 시 기본 기능 제공
