## 3. 화면별 상세 설계

### 3.1 [01-온보딩]

#### 개요
- **목적**: 신규 사용자의 직무와 경력 수준을 파악하여 맞춤형 추천 제공
- **관련 유저스토리**: UFR-USER-010
- **비즈니스 중요도**: 필수 (M) - 첫 인상 & 개인화 기반
- **접근 조건**: 첫 방문 사용자 (쿠키/세션 없음)

#### 주요 기능
1. 직무 선택 (4가지 중 택1)
2. 경력 수준 선택 (2가지 중 택1)
3. 선택 기반 스타터킷 추천

#### UI 구성요소

**레이아웃**
```
┌─────────────────────────────────┐
│        [로고]                    │
│                                  │
│   환영합니다!                     │
│   당신의 업무에 맞는 AI 리소스를   │
│   찾아보세요                      │
│                                  │
│   ┌────┐ ┌────┐ ┌────┐ ┌────┐  │
│   │마케터│ │개발자│ │디자이너│ │기획자│ │
│   └────┘ └────┘ └────┘ └────┘  │
│                                  │
│   경력 수준:                      │
│   ○ 3년 미만 (입문)               │
│   ○ 3년 이상 (심화)               │
│                                  │
│        [시작하기 →]               │
│                                  │
│        [건너뛰기]                 │
└─────────────────────────────────┘
```

**컴포넌트 상세**
- **Hero Section**: 서비스 소개 문구 (1-2줄)
- **직무 선택 카드**: 4개 카드, 아이콘 + 텍스트
  - 카드 크기: 150x150px (모바일: 100%)
  - 호버/선택 시: 테두리 강조, 그림자 효과
- **경력 선택**: 라디오 버튼 (2개)
- **CTA 버튼**: "시작하기" (Primary), "건너뛰기" (Secondary)

#### 인터랙션
1. 직무 카드 클릭 → 선택 상태 표시 (체크마크, 색상 변경)
2. 경력 수준 선택 → 라디오 버튼 활성화
3. "시작하기" 버튼 클릭:
   - 필수값(직무, 경력) 미선택 시 → 에러 메시지 (toast)
   - 선택 완료 시 → 로딩 (0.5초) → 대시보드로 이동
4. "건너뛰기" 클릭 → 대시보드 (기본 추천)

#### 상태 관리
- 선택값 저장: 로컬 스토리지 (7일 유효)
- API 전송: POST /api/users/onboarding

#### 반응형
- **Desktop (>1024px)**: 카드 4개 가로 배치
- **Tablet (768-1024px)**: 카드 2x2 그리드
- **Mobile (<768px)**: 카드 세로 스택

### 3.2 [02-로그인]

#### 개요
- **목적**: 이메일 기반 간편 로그인/회원가입
- **관련 유저스토리**: UFR-USER-020
- **비즈니스 중요도**: 필수 (M) - 리소스 다운로드 전제조건
- **접근 조건**: 다운로드 시도 or 직접 접근

#### 주요 기능
1. 이메일 입력
2. 매직 링크 발송 (비밀번호 없음)
3. 자동 로그인 처리

#### UI 구성요소

**레이아웃**
```
┌─────────────────────────────────┐
│            [로고]                │
│                                  │
│    간편하게 로그인하세요           │
│                                  │
│    ┌──────────────────────┐     │
│    │ 이메일 주소 입력      │     │
│    └──────────────────────┘     │
│                                  │
│    [로그인 링크 받기]             │
│                                  │
│    이메일로 전송된 링크를          │
│    클릭하면 로그인됩니다           │
│                                  │
│    또는                          │
│    [Google로 로그인]              │
│                                  │
│    [← 뒤로가기]                  │
└─────────────────────────────────┘
```

**컴포넌트 상세**
- **이메일 입력**: Input (type="email", placeholder="name@company.com")
- **CTA 버튼**: "로그인 링크 받기" (Primary)
- **소셜 로그인**: Google OAuth 버튼 (Optional)
- **안내 문구**: 이메일 인증 안내

#### 인터랙션
1. 이메일 입력 → 유효성 검증 (실시간)
2. "로그인 링크 받기" 클릭:
   - 유효하지 않은 이메일 → 에러 메시지
   - 유효한 이메일 → POST /api/auth/magic-link
   - 성공 → "이메일을 확인해주세요" 화면 전환
3. 매직 링크 클릭 (이메일) → 토큰 검증 → 자동 로그인 → 대시보드

#### 상태 관리
- 로그인 토큰: JWT (30일 유효)
- 세션 저장: HTTP-Only Cookie

#### 에러 처리
- 이메일 형식 오류: "올바른 이메일을 입력해주세요"
- 네트워크 오류: "네트워크 연결을 확인해주세요"
- 토큰 만료: "로그인 링크가 만료되었습니다. 다시 시도해주세요"

### 3.3 [03-대시보드]

#### 개요
- **목적**: 개인화된 추천 리소스와 최신 콘텐츠 제공
- **관련 유저스토리**: UFR-RCMD-010
- **비즈니스 중요도**: 필수 (M) - 서비스 메인 허브
- **접근 조건**: 로그인 후 기본 랜딩 페이지

#### 주요 기능
1. 개인화 추천 리소스 (5-10개)
2. 최신 리소스 (인기순)
3. Quick Access: 스타터킷/프롬프트/치트시트/실무사례
4. 검색 바 (상단 고정)

#### UI 구성요소

**레이아웃**
```
┌─────────────────────────────────────────────┐
│ [로고]  [검색바]    [알림] [프로필]         │ ← 헤더
├─────────────────────────────────────────────┤
│                                             │
│  안녕하세요, {닉네임}님! 👋                 │
│  오늘은 어떤 리소스를 찾으시나요?            │
│                                             │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐      │
│  │스타터킷│ │프롬프트│ │치트시트│ │실무사례│ │ ← Quick Access
│  └──────┘ └──────┘ └──────┘ └──────┘      │
│                                             │
│  ─── 당신을 위한 추천 ───                   │
│  [추천 이유 표시]                            │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐      │
│  │리소스1│ │리소스2│ │리소스3│ │리소스4│     │
│  └──────┘ └──────┘ └──────┘ └──────┘      │
│                                             │
│  ─── 최신 리소스 ───                        │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐      │
│  │리소스A│ │리소스B│ │리소스C│ │리소스D│     │
│  └──────┘ └──────┘ └──────┘ └──────┘      │
│                                             │
│  ─── 인기 실무사례 ───                      │
│  [사례 카드 3개]                             │
│                                             │
└─────────────────────────────────────────────┘
```

**컴포넌트 상세**

1. **헤더 (Global Navigation)**
   - 로고 (좌측)
   - 검색바 (중앙, 확장 가능)
   - 알림 아이콘 (우측)
   - 프로필 아이콘 (우측)

2. **Welcome Banner**
   - 사용자 닉네임
   - 인사말 (시간대별 변경)
   - 이모지

3. **Quick Access Cards**
   - 4개 카드 (스타터킷, 프롬프트, 치트시트, 실무사례)
   - 아이콘 + 텍스트 + 카운트
   - 호버 시 그림자 효과

4. **추천 섹션**
   - 섹션 타이틀 + "더보기" 링크
   - 리소스 카드 (가로 스크롤 또는 그리드)
   - 추천 이유 뱃지 (예: "마케터를 위한 추천")

5. **최신/인기 섹션**
   - 리소스 카드 (동일 디자인)
   - 정렬 옵션 (최신순/인기순)

**리소스 카드 (공통 컴포넌트)**
```
┌─────────────────┐
│   [썸네일]       │
│                  │
│  제목 (2줄 제한) │
│  ⭐ 4.5 (123)   │
│  💾 1,234 다운로드│
│  [태그] [태그]   │
└─────────────────┘
```

#### 인터랙션
1. Quick Access 카드 클릭 → 해당 섹션 페이지 이동
2. 리소스 카드 클릭 → 상세 페이지 이동
3. 검색바 포커스 → 검색 자동완성 드롭다운
4. "더보기" 링크 → 해당 카테고리 전체 목록

#### 상태 관리
- 추천 데이터: GET /api/recommend/personalized
- 최신 리소스: GET /api/resources/latest
- 사용자 정보: 전역 상태 (Zustand)

#### 성능 최적화
- 리소스 카드: Virtual Scroll (10개 이상)
- 이미지: Lazy Loading, WebP
- 추천 알고리즘: 서버 캐싱 (5분)

1. 직무별 필터링

| Mobile | < 768px | 1 | 16px |
| Tablet | 768 - 1023px | 2 | 24px |
| Desktop | 1024 - 1439px | 3-4 | 32px |
| Large Desktop | ≥ 1440px | 4-5 | 40px |

### 5.2 반응형 패턴

**1. 리소스 카드 그리드**
- Mobile: 1 column (세로 스택)
- Tablet: 2 columns
- Desktop: 3-4 columns

**2. 헤더**
- Mobile: 햄버거 메뉴 (사이드바)
- Tablet/Desktop: 전체 메뉴 표시

**3. 검색바**
- Mobile: 아이콘 → 클릭 시 전체 화면 검색
- Desktop: 상시 표시

**4. 필터**
- Mobile: 하단 시트 (Bottom Sheet)
- Desktop: 사이드바 or 상단 바

### 5.3 터치 최적화

- 버튼 최소 크기: 44x44px (iOS 권장)
- 터치 영역 간격: 최소 8px
- 스와이프 제스처: 카드 삭제, 좌우 네비게이션

## 8. 빈 상태 디자인 가이드 (Empty State)

### 8.1 빈 상태 원칙

빈 상태는 사용자가 아직 콘텐츠를 생성하지 않았거나 검색 결과가 없을 때 표시됩니다.
단순히 "데이터가 없습니다"가 아닌, **도움이 되는 안내와 다음 액션 제안**이 필요합니다.

**원칙:**
1. 긍정적이고 친근한 톤
2. 상황에 맞는 일러스트레이션
3. 명확한 다음 액션 제안
4. 도움말 또는 팁 제공

## 9. 에러 핸들링 & 피드백 메시지 가이드

### 9.1 에러 타입 및 처리

#### 에러 타입 1: 네트워크 오류

**원인**: 인터넷 연결 끊김, 서버 응답 없음

**메시지:**
```
😅 네트워크 연결을 확인해주세요
인터넷 연결이 불안정한 것 같아요.
잠시 후 다시 시도해주세요.

[다시 시도]
```

**처리:**
- Toast 알림 (5초)
- 자동 재시도 (3회)
- 실패 시 에러 메시지

#### 에러 타입 2: 인증 만료

**원인**: JWT 토큰 만료, 세션 종료

**메시지:**
```
🔐 로그인이 만료되었어요
다시 로그인해주세요.

[로그인하기]
```

**처리:**
- 모달 표시
- 로그인 페이지로 리다이렉트
- 이전 페이지 URL 저장 (로그인 후 복귀)

#### 에러 타입 3: 권한 부족 (403)

**원인**: 프리미엄 전용 기능 접근 (무료 사용자)

**메시지:**
```
💎 프리미엄 전용 기능이에요
이 기능은 프리미엄 회원만 사용할 수 있어요.

프리미엄 혜택:
• 모든 리소스 무제한 접근
• 워터마크 제거
• AI API 통합

[프리미엄 보기]  [나중에]
```

**처리:**
- 모달 표시
- 프리미엄 페이지로 이동 옵션

#### 에러 타입 4: 서버 오류 (500)

**원인**: 서버 내부 오류, DB 연결 실패

**메시지:**
```
😢 일시적인 문제가 발생했어요
서버에 문제가 생긴 것 같아요.
잠시 후 다시 시도해주세요.

문제가 계속되면 support@genai-info.com으로
문의해주세요.

[다시 시도]  [고객센터]
```

**처리:**
- Toast 알림
- 에러 로그 전송 (Sentry 등)

#### 에러 타입 5: 유효성 검증 실패 (400)

**원인**: 잘못된 입력값, 필수 필드 누락

**메시지 예시:**
- "이메일 형식이 올바르지 않아요"
- "제목을 입력해주세요 (최소 10자)"
- "파일 크기는 10MB 이하여야 해요"

**처리:**
- 인라인 에러 메시지 (필드 아래)
- 빨간색 테두리
- 포커스 이동

#### 에러 타입 6: 리소스 없음 (404)

**원인**: 삭제되었거나 존재하지 않는 리소스

**메시지:**
```
🤔 페이지를 찾을 수 없어요
요청하신 페이지가 존재하지 않거나
삭제되었을 수 있어요.
```

- 버튼 클릭 처리 ("저장 중...", "제출 중...")
- 부분 데이터 로딩 (댓글 더보기)
- 검색 자동완성

**디자인:**
```
[🔄 저장 중...]
```

**구현:**
```jsx
<button disabled={isLoading}>
  {isLoading ? (
    <>
      <Spinner size="small" />
      <span>저장 중...</span>
    </>
  ) : (
    <span>저장</span>
  )}
</button>
```

#### 타입 4: Progress Bar (진행률)

**사용 시기:**
- 파일 업로드
- 파일 다운로드
- 다단계 프로세스 (온보딩)

**디자인:**
```
다운로드 중... 45%
[▓▓▓▓▓▓▓▓▓░░░░░░░░░░]
```

**구현:**
```jsx
<div className="progress">
  <div className="progress__label">
    다운로드 중... {progress}%
  </div>
  <div className="progress__bar">
    <div 
      className="progress__fill"
      style={{ width: `${progress}%` }}
    />
  </div>
</div>
```

#### 타입 5: Pull to Refresh (당겨서 새로고침)

**사용 시기:**
- 모바일 리스트 페이지
- 대시보드

**디자인:**
```
   ↓
새로고침하려면 당기세요
```

### 10.2 로딩 상태 선택 플로우차트

```
로딩 상태 필요?
    |
    ├─ 전체 페이지? ──→ Full-Screen Loader
    |
    ├─ 리스트? ──────→ Skeleton Screen (6-10개)
    |
    ├─ 버튼 액션? ───→ Inline Spinner (버튼 내)
    |
    ├─ 파일 전송? ───→ Progress Bar
    |
    └─ 부분 영역? ───→ Inline Spinner or Skeleton
```

### 10.3 로딩 상태 UX 원칙

**1. 적절한 타임아웃**
- 2초 이내: 로딩 표시 없음 (즉시 표시)
- 2-5초: Skeleton or Spinner
- 5초 이상: Progress Bar + 예상 시간 안내

**2. 낙관적 UI (Optimistic UI)**
- 저장, 좋아요: 즉시 UI 업데이트 → 백그라운드 API 호출
- 실패 시: Rollback + 에러 메시지

**예시:**
```jsx
// 즐겨찾기 클릭
const handleFavorite = () => {
  // 1. 즉시 UI 업데이트
  setIsFavorited(true);
  
  // 2. API 호출
  api.addFavorite(id)
    .catch(() => {
      // 3. 실패 시 Rollback
      setIsFavorited(false);
      toast.error('즐겨찾기 추가 실패');
    });
};
```

**3. 의미 있는 로딩 메시지**
- "로딩 중..." (X)
- "프롬프트를 불러오는 중..." (O)
- "AI가 응답을 생성하는 중... (평균 10초)" (O)

**4. 스켈레톤 개수**
- 화면에 보이는 아이템 수만큼 표시 (6-10개)
- 너무 많으면 오히려 UX 저하
