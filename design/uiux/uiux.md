## 1. 프로젝트 개요

### 1.1 서비스 목표
**"업무 목적에 적합한 생성형 AI를 빠르게 찾고 싶은 직장인"**을 위한 생성형 AI 정보 제공 서비스

### 1.2 타겟 사용자
- **일반 사용자**: 직무별 AI 도구를 빠르게 활용하고 싶은 직장인 (마케터, 개발자, 디자이너, 기획자)
- **커뮤니티 기여자**: 성공 사례와 노하우를 공유하고 평판을 쌓고 싶은 사용자
- **프리미엄 사용자**: 고급 기능과 워터마크 제거를 원하는 파워 유저
- **기업 관리자**: 팀 라이선스를 구매하고 관리하는 담당자
- **팀원**: 기업 라이선스를 통해 프리미엄 혜택을 받는 직원

### 1.3 핵심 가치 제안
1. **빠른 시작**: 직무별 스타터킷으로 30분 안에 AI 도구 활용 시작
2. **검증된 리소스**: 실무자들이 검증한 프롬프트, 치트시트, 사례
3. **실무 적용 가능**: Before/After 데이터와 ROI 계산기 제공
4. **커뮤니티 기반**: 사용자 기여와 평가로 지속적으로 업데이트
5. **팀 협업 지원**: 팀 라이선스로 조직 전체의 AI 역량 강화

### 1.4 디자인 철학
1. **모바일 우선 (Mobile First)**: 언제 어디서나 빠르게 접근 가능
2. **명확한 정보 계층**: 중요한 정보를 먼저, 복잡한 내용은 접기/펼치기
3. **즉각적인 피드백**: 모든 액션에 명확한 시각적/촉각적 피드백 제공
4. **일관된 디자인 시스템**: 컴포넌트 재사용으로 학습 곡선 최소화
5. **점진적 공개**: 초보자도 쉽게, 전문가도 깊게 탐색 가능

**레퍼런스**: 뤼튼(WRTN)의 UI/UX 패턴 (https://wwit.design/2024/12/24/wrtn/)
- 카드 기반 레이아웃
- 바텀시트/모달 활용
- 로딩 상태 명확히 표시
- 입력 폼의 단계적 안내

## 4. 화면별 상세 설계

### 4.1 온보딩 - 직무선택 화면

**화면명**: `02-온보딩-직무선택`

#### **개요**
- **목적**: 신규 사용자가 직무와 경력을 선택하여 개인화된 추천을 받을 수 있도록 설정
- **관련 유저스토리**: UFR-USER-010
- **비즈니스 중요도**: Must Have (M/5)

#### **주요 기능**
1. 직무 선택 (필수): 마케터/개발자/디자이너/기획자
2. 경력 수준 선택 (필수): 3년 미만(입문) / 3년 이상(심화)
3. 선택 완료 시 홈 화면으로 이동

#### **UI 구성요소**

**헤더**
- 진행 상태 표시 (Step 1 of 2)

- 아이콘 + 라벨
- 탭 전환 시 페이드 애니메이션

### 5.2 화면 전환 패턴

1. **스택 푸시 (Stack Push)**
   - 사용: 상세 화면 진입
   - 애니메이션: 우측에서 슬라이드 인
   - 예: 목록 → 상세

2. **모달 (Modal)**
   - 사용: 작은 액션 (로그인, ROI 계산 등)
   - 애니메이션: 하단에서 슬라이드 업
   - 배경 딤 처리

3. **바텀시트 (Bottom Sheet)**
   - 사용: 옵션 선택, 공유 등
   - 애니메이션: 하단에서 슬라이드 업
   - 드래그로 닫기 가능

4. **페이드 (Fade)**
   - 사용: 같은 레벨 화면 전환
   - 애니메이션: 페이드 아웃 → 페이드 인

## 6. 반응형 설계 전략

### 6.1 브레이크포인트

- **Mobile**: 320px - 767px (기본, 모바일 우선)
- **Tablet**: 768px - 1023px
- **Desktop**: 1024px 이상

### 6.2 레이아웃 전략

**Mobile (기본)**
- 1열 레이아웃
- 풀 너비 카드
- 하단 탭 바 고정

**Tablet**
- 2열 그리드
- 사이드바 (선택)
- 하단 탭 바 유지

**Desktop**
- 3열 그리드
- 좌측 사이드바 네비게이션
- 우측 정보 패널 (선택)
- 하단 탭 바 → 좌측 사이드바로 전환

### 6.3 컴포넌트 반응형 규칙

**카드**
- Mobile: 100% 너비
- Tablet: 48% 너비 (2열)
- Desktop: 32% 너비 (3열)

**검색 바**
- Mobile: 전체 너비
- Desktop: 최대 600px (중앙 정렬)

**폰트 크기**
- Mobile: 기본 14px
- Desktop: 기본 16px

**여백**
- Mobile: 16px
- Tablet: 24px
- Desktop: 32px

## 7. 접근성 보장 방안

### 7.1 WCAG 2.1 AA 준수

1. **색상 대비**
   - 텍스트: 최소 4.5:1
   - 큰 텍스트 (18pt+): 최소 3:1
   - UI 컴포넌트: 최소 3:1

2. **키보드 접근성**
   - 모든 인터랙티브 요소 Tab으로 접근 가능
   - Focus 상태 명확히 표시
   - Escape로 모달/바텀시트 닫기

3. **스크린 리더**
   - 모든 이미지에 alt 텍스트
   - ARIA 레이블 사용
   - 헤딩 구조 명확히

### 7.2 키보드 네비게이션

- **Tab**: 다음 요소로 이동
- **Shift + Tab**: 이전 요소로 이동
- **Enter/Space**: 버튼 클릭
- **Escape**: 모달/팝업 닫기
- **Arrow Keys**: 목록 내 이동 (선택)

### 7.3 스크린 리더 지원

- **ARIA 레이블**: 모든 인터랙티브 요소
- **Live Region**: 토스트, 알림
- **Role 속성**: button, link, navigation 등
- **Alt Text**: 모든 의미 있는 이미지
