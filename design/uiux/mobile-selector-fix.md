# 모바일 셀렉터 박스 개선 작업

## 📋 문제 정의
모바일 화면에서 셀렉터 박스(select)의 글자가 너무 작고 위치가 맞지 않는 문제 발견

### 문제점
1. 글자 크기가 14px로 너무 작아 가독성 저하
2. iOS에서 16px 미만 폰트는 자동 줌이 발생하여 UX 저하
3. 터치 영역이 44px 미만으로 모바일 접근성 기준 미달
4. 네이티브 드롭다운 화살표가 텍스트와 겹쳐 보이는 문제

## 🎯 해결 방안

### 1. 폰트 크기 증가
- **변경 전**: `font-size: 14px`
- **변경 후**: `font-size: 16px`
- **근거**: iOS 자동 줌 방지 및 가독성 향상

### 2. 최소 높이 지정
- **변경 전**: `padding: 12px` (높이 약 38px)
- **변경 후**: `min-height: 48px`, `padding: 12px 16px`
- **근거**: 모바일 터치 영역 권장 사이즈 (최소 44px) 준수

### 3. 커스텀 드롭다운 아이콘
- **추가 사항**: 
  - `appearance: none` - 네이티브 스타일 제거
  - SVG 아이콘 추가 - 일관된 드롭다운 표시
  - `padding-right: 40px` - 아이콘 영역 확보
- **근거**: 크로스 브라우저 일관성 확보

### 4. 라인 높이 조정
- **추가 사항**: `line-height: 1.5`
- **근거**: 텍스트 가운데 정렬 및 가독성 향상

## 📝 수정된 파일 목록

### 1. common.css (전역 스타일)
```css
.input-field, select {
  width: 100%;
  min-height: 48px;
  padding: 12px var(--space-md);
  font-size: 16px;
  line-height: 1.5;
  /* ... 기타 속성 ... */
}

select {
  appearance: none;
  background-image: url('data:image/svg+xml;...');
  background-repeat: no-repeat;
  background-position: right 12px center;
  background-size: 20px;
  padding-right: 40px;
  cursor: pointer;
}
```
